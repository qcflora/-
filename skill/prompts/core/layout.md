# 核心规则：基线版式与导出约定

## 1. 页面设置（A4 标准）

所有公文及正式材料的默认页面设置为 A4 规格，**除非用户明确要求其他规格**。

| 参数 | 标准值 | 说明 |
|------|--------|------|
| 纸张大小 | A4（210mm × 297mm） | 纵向使用，特殊表格可横向 |
| 上页边距 | 37mm | 约 2.6 厘米（留出天头及发文机关标识空间） |
| 下页边距 | 35mm | 约 2.5 厘米（留出页脚及版记空间） |
| 左页边距 | 28mm | 约 2.0 厘米（装订侧） |
| 右页边距 | 26mm | 约 1.8 厘米 |
| 版心尺寸 | 156mm × 225mm | 正文实际排版区域 |
| 每页行数 | 22 行 | 固定值，行距 28-30 磅 |
| 每行字数 | 28 字 | 三号仿宋标准字间距 |

### 1.1 页面设置纪律
- 不得擅自调整页边距以压缩或扩充内容篇幅。
- 如内容超长需分页，应正常分页，不得通过缩小字号或行距来硬塞进一页。
- 附件表格如需横向排版，应在该页单独设置横向页面，前后恢复纵向。

## 2. 标题排版

### 2.1 大标题（发文机关标识 + 事由 + 文种）
- **字体**：二号小标宋体（方正小标宋或华文中宋）
- **字号**：22pt
- **颜色**：黑色（发文机关标识如为红头，按用户提供的红头模板处理）
- **对齐**：居中
- **位置**：红色分割线下，版心上部适当位置
- **换行**：标题较长需回行时，应词意完整、排列对称、长短适宜，推荐使用梯形或菱形排列

### 2.2 副标题与篇内标题
- **副标题**：如有副标题，用三号楷体，居中，前加破折号「——」
- **篇内一级标题**：黑体，三号，左对齐，前空一行
- **篇内二级标题**：楷体，三号，左对齐，前空半行
- **篇内三级及以下标题**：仿宋（可加粗），三号，左对齐

---

## 3. 页码设置

### 3.1 页码格式
- **字体**：四号宋体
- **字号**：14pt
- **位置**：页面底端外侧（单页右对齐，双页左对齐）
- **样式**：阿拉伯数字，不加装饰线，不加「第」「页」字样
- **格式**：`- 1 -` 或直接 `1`（按用户所在地区惯例，默认直接数字）

### 3.2 特殊页码规则
- **首页**：如为红头文件，首页一般显示页码；如为信函格式，首页可不显示。
- **附件页**：连续编排页码，不与正文断开。
- **封面/目录**：如含封面或目录，页码可单独编排（使用罗马数字），正文重新用阿拉伯数字。

## 4. Word 导出约定（docx.js）

当用户要求导出 Word 文档时，通过 `docx.js` 库生成，遵循以下约定：

### 4.1 字体映射表
在 docx.js 中，按以下映射设置字体：

| 排版元素 | docx.js 字体名称 | 后备字体 |
|----------|------------------|----------|
| 大标题 | FangSong / SimSun（小标宋需系统支持） | SimSun |
| 一级标题 | SimHei（黑体） | Arial Black |
| 二级标题 | KaiTi（楷体） |楷体_GB2312 |
| 正文/三级标题 | FangSong（仿宋） | FangSong_GB2312 |
| 页码 | SimSun（宋体） | Times New Roman |
| 英文/数字 | Times New Roman | — |

### 4.2 按文种自动套用版式方案

| 文种类别 | 版式特征 | docx.js 处理要点 |
|----------|----------|------------------|
| **红头文件**（通知、通报、报告等） | 红头 + 发文字号 + 红色分割线 | 需在页眉插入红头图片，设置红色分割线 |
| **信函格式**（函、部分通知） | 上端预留发文机关标识，无红头 | 上边距适当缩小，首行显示发文机关全称 |
| **纪要格式** | 眉首为「XX会议纪要」标识 | 标题居中，正文前显示会议时间、地点、参会人员 |
| **白头文件**（方案、计划、总结等） | 无红头，标题直接居中 | 按常规大标题排版 |
| **联合行文** | 多个发文机关并排 | 发文机关名称横排，主办机关在前 |

### 4.3 导出文件结构
生成的 `.docx` 文件应包含以下结构：

```
document
├── sections
│   ├── properties（页面设置：A4、页边距）
│   ├── headers（页眉：红头/发文机关标识）
│   ├── footers（页脚：页码）
│   └── children（正文内容段落）
└── styles（预定义样式：Heading1、Heading2、BodyText 等）
```

### 4.4 导出质量要求
- **兼容性**：生成的 docx 文件应在 Microsoft Word 2016 及以上版本、WPS Office、LibreOffice 中正常打开。
- **可编辑性**：导出的文档应为可编辑状态，非图片嵌入。
- **字体嵌入**：如使用特殊字体（如方正小标宋），应提示用户确保目标计算机已安装对应字体，否则可能回退为宋体。

## 5. Markdown 成稿结构要求

### 5.1 文件头（Front Matter）
每份 Markdown 成稿应包含标准化的文件头，便于后续解析和导出：

```markdown
---
title: "XX市人民政府关于加强安全生产工作的通知"
doc_type: "通知"
direction: "下行"
author: "[发文单位]"
date: "[日期]"
task_id: "task_20240704_143052"
version: "1.0"
source_materials:
  - "素材1：XX局《安全生产情况汇报》2024-06-15"
  - "素材2：XX会议《会议纪要》2024-06-20"
---
```

### 5.2 正文结构标记
在 Markdown 中，使用以下约定来标记公文结构，以便 docx.js 转换时正确识别样式：

| 公文元素 | Markdown 标记 | 转换后样式 |
|----------|--------------|------------|
| 大标题 | `# 标题内容` | 二号小标宋居中 |
| 一级标题 | `## 一、标题内容` | 黑体三号 |
| 二级标题 | `### （一）标题内容` | 楷体三号 |
| 三级标题 | `#### 1. 标题内容` | 仿宋三号加粗 |
| 四级标题 | `##### （1）标题内容` | 仿宋三号 |
| 正文段落 | 普通段落 | 仿宋三号，首行缩进 2 字符 |
| 主送机关 | `> 主送：XXX` 或单独段落 | 顶格，仿宋三号 |
| 附件说明 | `> 附件：1.XXX` | 左空两字 |
| 结尾用语 | 单独段落 | 左空两字 |
| 落款 | 右对齐段落 | 右对齐，仿宋三号 |

### 5.3 特殊标记约定
- **占位符**：使用 `<span style="color:red">[待核实]</span>` 或 Markdown 加粗 `**[待核实]**`
- **素材来源标注**：使用 Markdown 脚注 `[^1]`，在文末统一列出
- **敏感内容标记**：使用引用块 `> 【敏感提示】...`
- **冲突提示**：使用引用块 `> 【冲突提示】...`
- **合规批注**：使用 HTML 注释 `<!-- [error] 批注内容 -->`，预览时隐藏，导出时可选保留

## 6. 排版一致性检查清单

在导出前，应对照以下清单检查版式一致性：

- [ ] 页面为 A4，页边距符合标准（上37/下35/左28/右26mm）
- [ ] 正文字体为三号仿宋，行距 28-30 磅
- [ ] 一级标题黑体、二级标题楷体、三级及以下仿宋（可加粗）
- [ ] 大标题为二号小标宋体居中
- [ ] 页码为四号宋体，位于底端外侧
- [ ] 首行缩进 2 字符，全文统一
- [ ] 附件说明左空两字，附件名称换行时与首字对齐
- [ ] 落款右对齐，成文日期用阿拉伯数字
- [ ] 无多余空行、无孤行寡行（可通过 docx.js 设置 widow/orphan 控制）

---

> 本规则为 `official-doc-assembler` Skill 的版式基础，确保所有成稿在视觉呈现上符合公文规范。Markdown 结构约定同时服务于人类可读性与机器可转换性。
