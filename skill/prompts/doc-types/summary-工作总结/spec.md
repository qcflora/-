# 工作总结

## 写作规则

- **适用场景**：阶段性复盘、年度总结、专项工作总结。
- **行文方向**：一般不设主送单位；如作为正式报送件，可在标题下说明报送对象或报送日期。
- **语气要求**：陈述/回顾口吻，无祈使语；喜忧皆报，不夸大成绩、不隐瞒问题。
- **约束条件**：
  - 必给：总结周期、基本情况、主要做法与成效（带数据对比）、经验体会、存在问题、下一步打算。
  - 数据要可核实，不编造数量、增幅、机构名称或文件号。
  - 不流水账、不面面俱到无重点。
  - 作为正式报送件时可在标题下注报送对象或日期，一般不设主送、不请批（区别于"报告"）。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中，常见格式为"[年度/阶段][事项]工作总结"。
- **主送单位**：一般不设主送单位；如作为正式报送件，可在标题下说明报送对象或报送日期。
- **正文格式**：3号仿宋体，首行缩进2字符；"主要做法、取得成效、存在问题、下一步打算"宜分层表达。
- **特殊元素**：
  - 一级标题可用黑体突出各板块。
  - 数据、案例、问题和经验做法应分别成段，避免通篇套话。
- **与其他文种的区别**：总结重在回顾经验、查找问题；与报告相比，总结更灵活，一般不设主送；与汇报材料相比，总结重自我盘点，汇报更面向特定对象与主题。

## 模板

### 标准结构

```markdown
# [单位名称][时间范围][事项名称]工作总结

## 一、基本情况
[填写工作背景、时间范围和总体情况。]

## 二、主要做法
（一）[做法一]
[填写具体做法。]

（二）[做法二]
[填写具体做法。]

## 三、工作成效
[填写主要成果、数据和典型经验；未核实的数据应标注待核实。]

## 四、存在问题
[填写短板不足、风险隐患或制约因素。]

## 五、下一步打算
[填写下一阶段工作安排和改进措施。]

## 落款（可选）
[单位名称]
[日期]
```

### 结尾用语

- 工作总结一般不单独设置固定结尾套语。
- 常以"下一步将持续抓好贯彻落实"或"下一步打算"中的具体措施自然收束。

### 注意事项

- 罗列数字而无提炼是大忌，要用"结论句 + 硬数据 + 案例"的三明治写法提炼规律。
- 面面俱到无重点会使总结失去价值。
- 夸大成绩、隐瞒问题会降低材料可信度。
- 写成不主送上级却请批的"四不像报告"是常见错误。
- 组织顺序宜遵循"先事实、后概括、再提炼"；经验体会应建立在前文事实基础上，不空口总结。
- 除写成绩和经验外，还应区分"问题短板"和"原因分析、改进方向"；下一步打算宜与问题逐条对应。

### 子类型

1. **年度工作总结**：按业务板块横向切块，做法段写"重视程度—具体措施—实际效果"。
2. **阶段性工作总结**：聚焦某一特定阶段的工作，时间范围明确。
3. **专项工作总结**：围绕某一专项工作或活动进行复盘，针对性更强。
4. **个人/部门工作总结**：范围较小，可更具体地描述个人或本部门的职责履行情况。
