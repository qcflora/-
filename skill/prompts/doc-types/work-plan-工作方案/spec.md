# 工作方案

## 写作规则

- **适用场景**：专项行动、阶段性工作、制度落地、项目推进。
- **行文方向**：一般不设主送单位；如作为通知附件或专项文件，可在标题下补充适用范围。
- **语气要求**：务实可操作，每项任务都要回答"谁来做、做到什么程度、何时完成"。
- **约束条件**：
  - 必给：制定背景与依据、工作目标（量化＋时间节点）、主要任务（每项拆到目标/举措/责任/标准）、实施步骤、保障措施、责任分工。
  - 目标要可考核；任务要可操作，避免空泛。
  - 与通知配套关系：较长的工作方案常作为通知附件印发，方案本体不再写"特此通知"式部署祈使套话。
  - 责任清单、任务台账等信息较多时，可通过附件或附表承载。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中，常见格式为"[事项]工作方案"或"[事项]实施方案"。
- **主送单位**：一般不设主送单位；如作为通知附件或专项文件，可在标题下补充适用范围或执行期限说明。
- **正文格式**：3号仿宋体，首行缩进2字符；"总体要求、目标任务、实施步骤、保障措施"宜分层写清。
- **特殊元素**：
  - 正文中的时间节点和责任主体应单独成句或成条。
  - 如需呈现流程链条、施工路线、点位分布等，更适合在附件中加入流程图、示意图、路线图、时间轴图。
- **与其他文种的区别**：工作方案重在"怎么做"，是操作层面的具体安排；与通知相比，方案不直接下达执行指令，而是提供实施路径。

## 模板

### 标准结构

```markdown
# 关于开展[事项名称]的工作方案

## 一、制定背景
[填写制定背景、政策依据或现实需求。]

## 二、总体要求
[填写指导思想、基本原则或总体安排。]

## 三、工作目标
[填写目标任务、预期成效或时间要求。]

## 四、主要任务
（一）[任务一]
[填写具体工作内容、责任分工和完成标准。]

（二）[任务二]
[填写具体工作内容、责任分工和完成标准。]

## 五、实施步骤
[填写阶段安排、时间节点和推进节奏。]

## 六、保障措施
[填写组织领导、督导检查、资源保障、纪律要求等。]

## 附件（可选）
附件1：[流程图或示意图标题]
图1：[图片标题]
[图片描述]
说明：[说明文字]
```

### 结尾用语

- 工作方案一般不单独设置套话式结尾，通常以措施或附件收束。
- 如确需收尾，可用：**"请结合实际认真贯彻执行。"**（常放在配套通知中）

### 注意事项

- 措施空泛不可操作是最大忌讳，必须具体到责任、时限、标准。
- 目标无量化无时间节点会导致无法考核。
- 任务与保障脱节会使方案流于形式。
- 把方案写成通知式部署套话或在附件方案里重复部署语气。
- 照搬模板只换头改尾会失去针对性。
- 如需呈现流程链条、施工路线等，更适合在附件中加入流程图、示意图；只有明显属于技术手册或操作指南时，才进一步使用附录。

### 子类型

1. **专项行动方案**：针对某一专项行动制定的详细实施计划，目标明确、时间紧、任务重。
2. **制度落地方案**：用于推动某项制度或政策的落地实施，重在细化操作路径。
3. **项目推进方案**：围绕具体项目制定，包含里程碑节点、资源配置、风险管理等。
4. **试点工作方案**：用于某项工作的先行先试，包含试点范围、试点任务、评估标准等。
