# 通知

## 写作规则

- **适用场景**：发布、传达、转发事项，安排部署工作，任免人员等。
- **行文方向**：下行文为主，也可平行发送。
- **语气要求**：部署安排要具体明确，传达事项要简洁清晰，知照性内容要直截了当。
- **约束条件**：
  - 一文一事或一事一主题，避免内容庞杂。
  - 标题动词要准确：本机关对本机关形成的方案、办法、细则等正式下发时，优先用「印发」；批转下级机关文件时优先用「批转」；转发上级、同级或不相隶属机关文件时优先用「转发」，不要混用。
  - 正文与附件分工明确：通知正文重在传达、部署和提出要求；方案、办法、工作要点等实体内容较长时，优先作为附件随通知印发，正文从简，不把附件全文再抄写一遍。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。
  - 如用户已提供主送单位、发文单位或日期，则据实写入；未提供时保留占位符，不省略对应结构。

## 版式要求

- **标题格式**：2号小标宋体居中，常见格式为"关于[事项]的通知"。
- **主送单位**：标题下空一行顶格书写，末尾用全角冒号。高规格下行通知主送常是多行机关序列，不必收成单一机关。
- **正文格式**：3号仿宋体，首行缩进2字符；一级标题用黑体，二级标题用楷体。
- **特殊元素**：
  - 部署类通知宜把总体要求、重点任务、工作要求分层写清，报送时限和责任单位单独成句。
  - 印发型、批转型、转发型通知正文通常较短，可直接使用传达句式；部署型通知再完整展开任务和要求。
- **与其他文种的区别**：通知重在"知"和"行"，要有明确的执行要求；与决定相比，通知不用于重大决断；与意见相比，通知不用于提原则办法。

## 模板

### 标准结构

```markdown
# 关于[事项名称]的通知

## 主送单位
[主送单位]：

## 正文
为[目的或依据]，现就[事项名称]通知如下：

一、总体要求
[填写总体目标、工作背景或实施原则。]

二、重点任务
[填写具体工作安排、时间节点、责任要求。]

三、有关要求
[填写报送要求、组织要求、纪律要求。]

## 落款
[发文单位]
[日期]

## 附注
（联系人：[联系人] 联系电话：[固定电话]，[手机号]）

## 附件（可选）
附件1：[附件名称]
```

### 结尾用语

- 部署型、事务性通知常用：**"特此通知。"**
- 避免与开头"现将……通知如下"语义重复。

### 注意事项

- 转发标题不要层层叠加冗长。
- 不要把通知写成长篇背景材料。
- 该用意见（提原则办法）或决定（重大决断）的事项不要误用通知。
- 需要说明办理流程、任务链条时，可在附件中加入流程图、示意图；正文一般只写"详见附件"。
- 本项目默认保留附注联系人信息；用户明确要求删除时再去掉。

### 子类型

1. **印发型通知**：用于印发本机关制定的方案、办法、细则等。正文简短，常用"现印发给你们，请结合实际认真贯彻执行"。
2. **批转型通知**：用于批转下级机关的文件。正文常用"现批转给你们，请认真抓好落实"。
3. **转发型通知**：用于转发上级、同级或不相隶属机关的文件。正文常用"现转发给你们，请认真贯彻执行"。
4. **部署型通知**：用于安排部署工作。正文需完整展开总体要求、重点任务、工作要求等内容。
5. **事务性通知**：用于任免人员、机构调整、会议通知等日常事务。内容简洁直叙。
