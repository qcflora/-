# 纪要

## 写作规则

- **适用场景**：记载会议主要情况和议定事项。
- **行文方向**：根据会议性质和参会范围确定，可为内部下发或平行抄送。
- **语气要求**：严谨、准确、精炼、便于执行和操作；以"会议"为主语统领，提炼升华而非逐字记录。
- **约束条件**：
  - 必给：会议名称、时间、地点、主持人、出席/列席人员、议题。
  - 必给：会议议定事项（决定了什么、谁负责、何时完成）与后续要求。
  - 区分"会议认为/指出/强调/要求"各自要承载的内容。
  - 未达成一致的事项不写入议定事项，可写"继续研究""补充完善后另报"。
  - 纪要不是逐字会议记录，不机械照搬个人原话，不罗列无结论的来回讨论。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中。机关纪要常取"会议名称＋纪要"合成式（如"第X次常务会议纪要""[事项]专题会议纪要"），一般不套"关于……的纪要"。
- **会议信息**：文首列出时间、地点、主持人、出席、参加、记录等信息；人员信息宜集中排在正文前，不宜散插进正文各段。
- **正文格式**：3号仿宋体，首行缩进2字符；"会议认为，……"和"会议决定，……"一般直接作为正文提示语连续书写，不另设同名小标题。
- **特殊元素**：
  - "会议决定，……"后的事项建议逐条编号，责任单位、完成时限单独表达，便于执行和跟踪。
  - 多议题纪要的正文层级宜保持统一：要么按议题编号组织，要么按"会议认为/会议决定/责任分工"组织。
- **与其他文种的区别**：纪要与会议记录不同，纪要是提炼后的结论性文书；纪要不要写成汇报材料或宣传报道。

## 模板

### 标准结构

```markdown
# [会议名称]纪要

## 会议基本信息
- 会议名称：[会议名称]
- 时间：[时间]
- 地点：[地点]
- 主持人：[主持人]

## 出席
[出席人员]

## 参加
[参加人员]

## 请假（可选）
[请假人员]

## 记录
[记录人]

[导语段，交代会议围绕什么事项召开、听取了什么、研究了什么、审议了什么。]

会议认为，[概述会议对当前情况的判断。]

会议指出，[重要观点与基本情况。]

会议强调，[重点重申的定调。]

会议要求，[可执行的任务、责任与时限。]

会议决定，一、[议定事项一]
[补充说明。]

二、[议定事项二]
[补充说明。]

## 责任分工
[责任单位]于[时间]前完成[任务]。

## 后续要求
[跟踪督办、报送反馈、保密要求等。]

## 附件（可选）
1. [附件名称]

## 版记
主送：[主送单位]
抄送：[抄送单位]
审核：[审核人]
```

### 结尾用语

- 纪要一般不写寒暄式结尾，直接收束到责任和要求。
- 议定事项写完即收束，一般不写"特此"类寒暄结尾。

### 注意事项

- 写成会议记录（原始逐字存档）是常见错误。
- 把未达成一致的事项写进议定事项会造成执行歧义。
- 流水账式写法会失去纪要的价值。
- 使用第一人称是错误，应以"会议"为主语。
- 标题堆叠多个中心事项会使纪要主题不清。
- 议定事项空泛、无办理主体与时限会导致无法执行。
- 多议题会议宜按照会议实际议题顺序组织正文，一事一段、一事一条，不同议题不要交叉混写。
- 对外公开与内部使用口径要区分：公开发布宜压缩内部流程、争议意见和敏感细节。

### 子类型

1. **常务会议纪要**：适用于政府、党组、党委等常务会议，标题常见为"第[次数]次常务会议纪要"；语言要稳、硬、概括性强。
2. **办公会议纪要**：适用于机关内部办公会议，重在协调事项、任务安排和责任落实，语言更直接、更执行化。
3. **专题会议纪要**：适用于围绕某一项重点工作召开的专题研究会，标题常见为"[事项]专题会议纪要"；重在问题分析、研究意见、会议决定。
4. **联席/协调/会商会议纪要**：适用于多部门共同研究协同事项，重在各方共识、职责边界、配合机制和时间节点。
5. **座谈/研讨会议纪要**：适用于调研座谈、征求意见等场景，重在归纳意见、提炼共识，不宜误写成已经形成刚性决定的执行纪要。
