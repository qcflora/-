# 汇报材料

## 写作规则

- **适用场景**：向领导汇报、向检查组报送、向上级报备。
- **行文方向**：常见事务材料（非法定文种），可面向特定汇报对象与主题裁剪板块。
- **语气要求**：喜忧皆报、报够而不揽功、报透而不推卸；用观点统率材料，先亮观点（结论）再铺数据。
- **约束条件**：
  - 必给：汇报对象与场合、基本情况、主要做法与成效（带数据）、存在问题、下一步打算、需上级协调或拍板的事项。
  - 不报喜不报忧、不堆数字无结论。
  - 把需领导/调研组关注、协调、拍板、批示的事项单独拎出，不埋在一般进展里。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中，也可按汇报场景采用更简洁的黑体标题。常见格式为"关于[事项名称]的汇报材料"。
- **主送单位**：一般不设主送单位；如明确报送对象，可在标题下以副题或说明行体现（如"——在××座谈会上的汇报"）。
- **正文格式**：3号仿宋体，首行缩进2字符；"基本情况、工作进展、问题不足、下一步打算"宜分层呈现。
- **特殊元素**：
  - 面向领导阅示的材料应多短段落、少长句，关键数据、问题和建议可单独成段突出。
  - 较适合在附件中增设现场照片、数据图表、流程图、对比图、架构图等图片说明。
- **与其他文种的区别**：汇报材料比报告/工作总结更灵活，可面向特定汇报对象与主题裁剪板块；与工作总结相比，汇报更面向特定对象、突出对方关注点并单列需协调事项。

## 模板

### 标准结构

```markdown
# 关于[事项名称]的汇报材料

## 一、工作进展
[填写工作推进情况、阶段安排和主要节点。]

## 二、主要成效
[填写取得的成效、典型经验和阶段成果。]

## 三、存在问题
[填写当前短板不足、堵点难点和风险问题。]

## 四、下一步措施
[填写下一步工作安排、改进措施和需要协调事项。]

## 五、需要说明事项（可选）
[填写需补充说明或请领导关注的事项。]

## 落款（可选）
[汇报单位]
[日期]

## 附件（可选）
附件1：[图片或图表标题]
图1：[图片标题]
[图片描述]
说明：[说明文字]
```

### 结尾用语

- 常用：**"以上汇报，不妥之处，请批评指正。"**
- 或：**"以上汇报，请审示。"**
- 也可直接用"现将有关情况汇报如下"引出主体，以措施或建议自然收束。

### 注意事项

- 报喜不报忧会失去汇报材料的价值。
- 罗列材料无观点会让阅读者无法快速把握核心。
- 不研究汇报对象的关注点会导致汇报缺乏针对性。
- 把关键决策点埋在一般工作进展中会导致重点不突出。
- 层次清楚但不宜过多，逻辑强但不绕，口语化但不粗。
- 先说结果再说过程，一事一段，不重复。
- 转 PPT/讲话稿时先理清结构再场景化。

### 子类型

1. **工作汇报型**：通用式，基本情况 → 主要做法成效 → 存在问题 → 下一步打算（及需协调事项）。
2. **情况反映型**：侧重反映某一问题或情况，基本情况 + 主要成效 + 突出问题。
3. **经验总结型**：侧重总结经验和启示，主要做法 + 成效 + 启示。
4. **问题揭示型**：侧重揭示问题和提出对策，存在问题 + 原因分析 + 对策。
5. **专题汇报型**：面向特定专题/调研主题，直接按专题分块铺排。
