# 简报

## 写作规则

- **适用场景**：简要报送动态、会议情况、阶段成果、当前新闻整理。
- **行文方向**：属信息类文书、非法定公文，正式行文时常作报告或函的附件。
- **语气要求**：第三人称客观陈述、动词带头、嵌入具体数字，观点鲜明；新闻笔法、强时效。
- **约束条件**：
  - 必给：简报名称与期号、编发单位、核心事实（时间/主体/事件/概括评价）、要点。
  - 短小精悍、强时效，单期常以 2000 字内为度。
  - 一事一报或综合提炼，不把多个无关事项硬拼成同一篇简报。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中，或与本单位简报模板一致的标题字体。报名归版头，正文标题专讲事，不在标题里重复套"××简报"。
- **版头信息**：可在标题下增设期数、编发单位、日期等信息行；期号用"〔年份〕第 X 期"或"第 X 期"。
- **正文格式**：3号仿宋体，首行缩进2字符；新闻动态型简报宜短段落、快节奏、少空话。
- **特殊元素**：
  - 小标题建议用黑体或楷体，信息点较多时可采用"一、（一）"或项目符号式分段。
  - 较适合通过附件、附图或配套图表承载照片、截图、传播热度图、现场图和流程图。
- **与其他文种的区别**：简报偏常态连续编号、面向相对固定阅知范围、笔法新闻化；与报告相比，简报不是法定上行文；与情况专报相比，简报针对动态信息，情况专报针对单一专门事项定向深挖。

## 模板

### 标准结构

```markdown
# [事项名称]简报

## 一、导语
[用一段话概括时间、地点、主体和主要情况。]

## 二、主要信息
（一）[信息点一]
[填写事实摘要。]

（二）[信息点二]
[填写事实摘要。]

## 三、工作建议或提示
[填写后续关注事项、工作提示或简要建议。]

## 报送范围（可选）
[填写报送对象或阅知范围。]

## 附件（可选）
附件1：[图片或图表标题]
图1：[图片标题]
[图片描述]
说明：[说明文字]
来源：[来源]
```

### 结尾用语

- 简报非公文，不固定套话。
- 以提示事项、工作建议、下一步或报送范围收束，亦可刊物化处理或省略。

### 注意事项

- 冗长失时效会丧失简报价值。
- 堆砌材料无观点会沦为"材料仓库"。
- 标题平淡、导语不会用新闻笔法会影响阅读效果。
- 与报告/情况专报混淆是常见错误。
- 内部简报宜坚持"一期一主题""一报一重点"。
- 地方或系统样式中的"第[期]期"、编发单位、编发日期以及"报：/送：/发："等尾部信息，是否保留应以用户模板或系统习惯为准。

### 子类型

1. **动态简报**：报道某一工作或活动的最新进展，时效性强。
2. **会议简报**：快速报道会议主要情况和要点，通常当日或次日编发。
3. **经验简报**：总结提炼某项工作的典型经验和做法，供学习借鉴。
4. **综合简报**：围绕某一主题综合多条信息，信息密度高。
