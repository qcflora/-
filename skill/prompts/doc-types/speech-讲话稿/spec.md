# 讲话稿

## 写作规则

- **适用场景**：领导讲话、会议发言、动员部署、总结点评。
- **行文方向**：面向特定场合和对象，一般不设主送单位。
- **语气要求**：书面为主、口语为辅，庄重而有感染力；观点句、分项标题对仗工整、提气有力；有现场感、第一人称、动员色彩。
- **约束条件**：
  - 必给：讲话场合、面向对象、主题与目的、讲话人身份。
  - 必给：要传达的认识（怎么看）、部署的任务（怎么办）、提出的要求（怎么干）。
  - 多用"我们""我讲几点""我强调"，句子宜短、少嵌套，文件号与长书面引文应压缩，便于口头宣读。
  - 篇幅与体例：会场总结/部署讲话约 2000—3000 字常见，分项 3—5 条；致辞、主持词等更短。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。

## 版式要求

- **标题格式**：2号小标宋体居中，或按会场材料习惯使用黑体标题。常见格式为"在[会议或活动名称]上的讲话"。
- **称谓**：一般不设主送单位，开头如有称谓，可顶格单独成段并加全角冒号，且全篇通常只出现一次（如"同志们：""各位代表："）。
- **正文格式**：3号仿宋体，首行缩进2字符；"一、二、三"式层级结构较常见，重点句可适度单列。
- **特殊元素**：
  - 讲话稿更强调口头表达的节奏，可使用较短段落和提示性过渡句。
  - 可在层次之间用重复称谓或提示性过渡句分隔。
- **与其他文种的区别**：讲话稿区别于报告/工作方案的核心是**有现场感、第一人称、动员色彩**；写成无人称的报告/方案会丢失第一人称与动员色彩。

## 模板

### 标准结构

```markdown
# 在[会议或活动名称]上的讲话

## 称谓
[各位领导、同志们：]

## 一、开场说明
[填写会议背景、讲话目的或总体评价。]

## 二、形势判断
[填写当前形势、问题分析或背景引入。]

## 三、重点任务
（一）[任务一]
[填写重点工作。]

（二）[任务二]
[填写重点工作。]

## 四、工作要求
[填写组织要求、纪律要求或落实要求。]

## 五、结束语
[填写总结性收束语或号召。]
```

### 结尾用语

- 动员型常以号召句提气：**"让我们更加紧密地团结在……周围，为……而努力奋斗！"**
- 宣读稿可缀：**"谢谢大家！"**
- 工作部署型可用：**"我就讲这些，请大家认真抓好落实。"**

### 注意事项

- 空话套话口号化（"正确的废话"）是大忌。
- 偏离主旨与领导意图会丧失讲话稿价值。
- 层级滥用、逻辑跳跃会影响表达效果。
- 写成无人称的报告/方案，丢了第一人称与动员色彩。
- 数据为论点服务，不罗列数字。
- 要先区分讲话、发言、致辞、主持词等具体场景；同样是会场材料，动员讲话、总结讲话、致辞和主持串词的语气、称谓和长度差异明显，不宜套一个模板。

### 子类型

1. **动员部署讲话**：以"一要、二要、三要"的祈使排比展开，重在鼓劲、部署、提要求。
2. **总结点评讲话**：先总评会议或工作，再讲成绩与不足，最后提要求。
3. **致辞/贺词**：更简短，礼仪性强，重在表达祝愿、感谢和期待。
4. **主持词**：串联会议议程，承上启下，语言简洁明快。
5. **表态发言**：代表某单位或个人表态，重在表达决心和具体措施。
