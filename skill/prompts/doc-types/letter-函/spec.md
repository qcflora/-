# 函

## 写作规则

- **适用场景**：平行机关或不相隶属机关之间商洽、询问、答复事项，请求批准。
- **行文方向**：平行或不相隶属机关之间。
- **语气要求**：身份平等，称"贵单位／你们"，既不居高临下也不仰视，措辞礼貌得体；商洽用"商请""拟请"，不用命令式部署口吻。
- **约束条件**：
  - **一函一事**，事项清楚。
  - 答复类先给来文标题与发文字号。
  - 主送可为单一机关，也可为多行平行/不相隶属机关序列，按从中央到地方、从主到次排列。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。
  - 如用户已提供主送单位、发函单位或日期，则据实写入；未提供时保留占位符，不省略对应结构。

## 版式要求

- **标题格式**：2号小标宋体居中，宜写成"关于[事项]的函"。
- **主送单位**：标题下空一行顶格书写，末尾用全角冒号。
- **正文格式**：3号仿宋体，首行缩进2字符；函文通常篇幅较短，事项、意见、请求或答复应直截了当。
- **特殊元素**：
  - 如需分点，可按常规层级标题处理；篇幅短时也可不分层，保持一事一函的紧凑结构。
  - 商洽、询问类函宜把反馈事项、反馈方式和反馈时限单独成句写清。
  - 答复类函则应优先把答复结论置前，不让对方通篇寻找结论。
- **与其他文种的区别**：隶属关系请示用"请示"，下行部署用"通知"，不相隶属或平行机关之间则用函。

## 模板

### 标准结构

```markdown
# 关于[事项名称]的函

## 主送单位
[主送单位]：

## 一、事项缘由
[说明来函或发函背景、目的和需要协商或答复的事项。]

## 二、主要意见
[写明商洽意见、请求事项或答复内容。]

## 三、配合事项
[如需对方反馈、办理或协同，写明具体要求和时限。]

## 落款
[发函单位]
[日期]

## 附注
（联系人：[联系人] 联系电话：[固定电话]，[手机号]）
```

### 结尾用语

- 商洽/请求型：**"特此函达。"**、**"特此函商。"**、**"请予支持为盼。"**
- 复函：**"特此函复。"**
- 提案、建议答复型可写：**"感谢您对……工作的关心和支持。"**

### 注意事项

- 不要把本应用函的事项升格为请示或通知。
- 一函多事是大忌。
- 答复型不要把结论埋在文末让对方通篇寻找。
- 本项目默认保留附注联系人信息；用户明确要求删除时再去掉。

### 子类型

1. **商洽函**：用于商洽工作、请求协助。语气礼貌，把请求事项、所需配合、反馈方式与时限写清。
2. **询问函**：用于询问有关政策、情况或征求对方意见。
3. **请批函**：向不相隶属机关请求批准有关事项。用"拟请""商请"等措辞。
4. **答复函**：答复对方来函询问或请求。先引来文标题与文号，再给出结论，结论置前。
