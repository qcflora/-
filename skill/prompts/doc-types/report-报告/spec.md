# 报告

## 写作规则

- **适用场景**：向上级汇报工作、反映情况、回复询问。
- **行文方向**：上行文，一般只写一个主送机关。
- **语气要求**：实事求是、喜忧皆报，不卑不亢，不夹带请求批准事项。
- **约束条件**：
  - 核心是"报告情况"，不要把请批事项混进正文主线。
  - 确需审批的，应改判为请示或请示、报告分写。
  - 成效尽量带可核实的数据与案例；没有就留占位、标待核实。
  - 制式/年度报告按主管部门规定的固定栏目顺序铺排，常含统计数字与制式表格。
  - 事实要求：不得编造未提供或未核实的事实、数据、时间、地点、机构名称、人员身份、文件号和政策依据；信息不足时标注"待核实"或保留占位符。
  - 如用户已提供主送单位、发文单位或日期，则据实写入；未提供时保留占位符，不省略对应结构。

## 版式要求

- **标题格式**：2号小标宋体居中，常见格式为"关于[事项]的报告"。
- **主送单位**：标题下空一行顶格书写，上行文一般只写一个主送机关更稳妥，末尾用全角冒号。
- **正文格式**：3号仿宋体，首行缩进2字符；"基本情况、工作进展、存在问题、下一步打算"宜分层表达。
- **特殊元素**：
  - 二级标题下优先采用"概述 + 一是、二是、三是"的展开方式。
  - 成效用"结论句 + 硬数据 + 必要案例"，不堆砌口号。
- **与其他文种的区别**：报告重在汇报，不应把请示事项混写进正文；与工作总结相比，报告是法定上行文，格式更严，常设主送单位。

## 模板

### 标准结构

```markdown
# 关于[事项名称]的报告

## 主送单位
[主送单位]：

## 一、基本情况
[交代事项背景、总体情况和报告缘由。]

## 二、工作开展情况
[分点写明已完成工作、主要做法和阶段成效。]

## 三、存在问题
[如实说明当前问题、风险或困难。]

## 四、下一步建议
[说明下一步工作安排或需要关注的事项。]

特此报告。

## 落款
[发文单位]
[日期]

## 附注
（联系人：[联系人] 联系电话：[固定电话]，[手机号]）

## 附件（可选）
附件1：[图表或照片标题]
```

### 结尾用语

- 一般报告常用：**"特此报告。"**
- 向人大/常委会审议用：**"以上报告，请审议"**、**"请予审查"**
- 制式/年度报告可不用套语，以数据或事项句自然收束。
- 切勿误用"请批示"等请示语气。

### 注意事项

- 把需批准事项写成报告会导致被搁置。
- 结尾误用"请批示"语气会混淆为请示。
- 制式年报误套缘由式起笔或漏掉规定栏目。
- 答复询问类报告宜围绕所问事项逐条回应，不机械套用全套骨架。
- 本项目默认保留附注联系人信息；用户明确要求删除时再去掉。

### 子类型

1. **上行工作报告**：向上级定期汇报工作进展，常用"特此报告"结尾。
2. **专项情况报告**：针对某一专项工作或突发事件进行汇报。
3. **答复询问报告**：围绕上级询问的事项逐条回应，开头常引来函。
4. **会议审议报告**：向人大、常委会或专题会议提交审议，结尾用"请审议"。
5. **制式/年度报告**：按固定栏目顺序铺排，如政务公开年报、审计年报等，依据式起笔、固定栏目、统计占位。
